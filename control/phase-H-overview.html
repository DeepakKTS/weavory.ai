§1 What weavory is

weavory.ai is an MCP server that lets AI agents share signed beliefs instead of calling each other directly. Every agent writes a belief (a signed subject + predicate + object statement) into a shared store. Any other agent can search those beliefs, subscribe to patterns, attest to who they trust, or tombstone stale ones. The server is the single source of coordination truth; agents are just writers and readers.

Public API surface — five tools, locked
Tool What it does
weavory.believe Write a signed belief (Ed25519 + BLAKE3-chained audit).
weavory.recall Search beliefs by query / signer / time / subscription queue. Supports as_of, merge strategies, conflict surfacing, reputation aggregation.
weavory.subscribe Open a pattern-based match queue; drain later via recall(subscription_id).
weavory.attest Raise or lower trust for a (signer, topic) pair.
weavory.forget Tombstone a belief (OR-set remove). Historical as_of queries still see it.

Everything else — subscriptions, merge strategies, trust gates, branches, escrow, reputation, incident replay — is server-side behavior that rides on those five primitives. Phase G added a lot of capability without adding a single new tool.

§4 Key tech you're running on

Everything in the repo is boring-on-purpose: no daemons, no native bindings, no bespoke binary formats. Exactly one pnpm install from green-field to Gate 7.

Layer Tech Why
MCP server @modelcontextprotocol/sdk + Zod Official protocol, schema-first.
Storage (today) In-memory Maps + array audit chain Gate-1 friendly, zero-daemon. LanceDB + DuckDB are planned backlog (B-0001/B-0002).
Crypto @noble/ed25519 + BLAKE3 Audited, pure JS, no native deps.
Signing Ed25519 over canonical JSON NANDA AgentFacts–compatible.
Audit BLAKE3 hash chain (tamper-evident) Any mutation breaks every subsequent hash.
Merge OR-set tombstones + pluggable LWW / consensus Deterministic; LLM-judge opt-in planned post-G.
Runtime Node 20+ (22 LTS in CI), TypeScript strict No any in src/.

§5 Each arena in two sentences

Phase G added five arena surfaces without adding a single MCP tool. Everything below rides on the five primitives.

Commons

Gate ✓
Many agents can coordinate without direct RPCs.
They publish beliefs; a subscriber's queue captures matches; consensus merge picks the trust-weighted winner when signers disagree.

Wall

Gate ✓
Adversaries can't corrupt the shared truth.
Adversarial mode raises the trust floor; low-trust signers are filtered from default recall; any audit-chain mutation fires an alarm into runtime.json and the incident is exportable.

Gauntlet

Gate ✓
You can explore "what if" strategies without breaking the main timeline.
cloneState makes a detached deep copy of the engine; branch and main diverge; as_of recall rewinds time; weavory replay rehydrates any incident from disk.

Bazaar

Gate ✓
Agents can trade capabilities and settle with cryptographic proof.
A capability offer + payment + delivery + settlement are four beliefs linked by causes[]; escrowStatus walks the chain; trust-aware reputation gates discovery.

Throne

Integration · Gate ✓
All four arenas coexist on one server.
One EngineState with adversarialMode: true ran Commons + Wall + Gauntlet + Bazaar simultaneously — queue delivered 3, trust gate filtered mallet, branch diverged from main, four-stage escrow settled accepted — and the final audit.verify() was still green.
